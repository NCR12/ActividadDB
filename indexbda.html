Investiga y desarrolla

  1. ¿Qué es un Dato?

    2. El dato es la representación de una variable cualitativa o cuantitativa. De ese modo, se le asigna un número, letra o símbolo.

  2. ¿Qué es información?

    3. La información es el conjunto de datos, procesados y ordenados para su comprensión, que aportan nuevos conocimientos. Puede ser a un individuo o sistema y sobre un asunto, materia, fenómeno o ente determinado.

  3. ¿Diferencia entre dato e información?

    4. La diferencia clave es que los datos son hechos brutos sin procesar, mientras que la información es el resultado de procesar y dar contexto a esos datos, convirtiéndolos en algo útil y con significado.

  4. ¿Tipos de datos?

    5. Los tipos de datos hacen referencia al tipo de información que se trabaja, donde la unidad mínima de almacenamiento es el dato, también se puede considerar como el rango de valores que puede tomar una variable durante la ejecución del programa.

    En programación podemos tener varios tipos de datos, pero los más conocidos son: numéricos, cadenas de caracteres, booleanos y sin valor.

    Numéricos

    Representan a los datos de tipo numérico. Estos se pueden subdividir en:

    • Integer: número sin decimales. Por ejemplo: 50; 70; 300.
    • Float: número con decimales. Por ejemplo: 5.3; 4.7; 8.24.
    • Double: número con decimales, similar a float, pero con capacidad de almacenar más decimales. Especialmente útil para variables o constantes que requieren valores con mayor precisión, por ejemplo el número PI que tiene muchos decimales.

    Cadenas de caracteres

    Quiere decir que almacenaremos texto. Tenemos dos grupos:

    • Char: es un solo carácter que puede ser una letra, número o un símbolo como por ejemplo #, @, %, a o 5.
    • String: es una cadena de más de un carácter como por ejemplo “¡Hola, Mundo!”

      • Boolean

      Es decir que algo es verdadero (true) o falso (false).

      Sin valor

      Sirve para expresar que una variable no tiene valor (Null) o que está no definido (Undefined).

  5. ¿Qué es un modelado de Datos?

    6. El modelado de datos es el proceso de crear una representación visual de un sistema de información o una base de datos que captura las relaciones entre diferentes elementos de datos. Este proceso ayuda a estructurar y organizar los datos de manera que sean fácilmente accesibles y comprensibles. Los modelos de datos se utilizan para diseñar bases de datos y sistemas de información y pueden incluir diagramas, entidades, atributos y relaciones.

  6. Tipos de bases de Datos, según:

    • Variabilidad:

    1. Estáticas:
      • Bases de datos que no cambian con el tiempo una vez creadas.
      • Ejemplo: Bases de datos históricas.
    2. Dinámicas:
      • Bases de datos que se actualizan constantemente.
      • Ejemplo: Bases de datos transaccionales.

    • Contenido:

    1. Bibliográficas:
      • Contienen información sobre documentos.
      • Ejemplo: Catálogos de bibliotecas.
    2. Texto completo:
      • Almacenan textos completos de documentos.
      • Ejemplo: Bases de datos de artículos científicos.
    3. Numéricas:
      • Contienen datos numéricos.
      • Ejemplo: Bases de datos de investigación científica.
    4. Multimedia:
      • Almacenan datos en forma de imágenes, audio, video.
      • Ejemplo: Bibliotecas digitales de multimedia.

    • Modelo de base de datos

    1. Relacionales:
      • Organizan los datos en tablas.
      • Ejemplo: MySQL, PostgreSQL.
    2. Jerárquicas:
      • Organizan los datos en una estructura de árbol.
      • Ejemplo: IBM Information Management System (IMS).
    3. Red:
      • Similar a la jerárquica pero permite relaciones más complejas.
      • Ejemplo: IDMS (Integrated Database Management System).
    4. Orientadas a objetos:
      • Los datos se almacenan como objetos, similar a la programación orientada a objetos.
      • Ejemplo: db4o.
    5. Documentales:
      • Almacenan datos en formato de documentos, usualmente JSON o XML.
      • Ejemplo: MongoDB.

  7. ¿Qué es el modelo relacional?

    8. El modelo relacional es un enfoque para gestionar bases de datos en el que los datos se organizan en tablas (o relaciones) que pueden ser manipuladas utilizando un lenguaje de consulta estructurado (SQL). Cada tabla contiene filas (tuplas) y columnas (atributos), y cada fila representa un registro único. Las tablas pueden estar relacionadas entre sí mediante el uso de claves primarias y foráneas.

  8. ¿Describa los elementos del Diagrama entidad Relacional?

    9. Los elementos principales de un Diagrama de Entidad-Relación (ERD) son:

    • Entidades: Representan objetos o conceptos que tienen una existencia independiente y sobre los cuales se necesita almacenar información. Se dibujan como rectángulos en el diagrama.
    • Atributos: Son las propiedades o características de las entidades. Se dibujan como óvalos conectados a sus respectivas entidades. Los atributos pueden ser simples, compuestos, derivados o multivaluados.
    • Relaciones: Representan las asociaciones entre dos o más entidades. Se dibujan como rombos o líneas conectadas a las entidades que participan en la relación. Las relaciones pueden ser uno a uno, uno a muchos, o muchos a muchos.
    • Llaves Primarias (Primary Keys): Son atributos que identifican de manera única a una entidad dentro de un conjunto de entidades. Se subrayan en el diagrama.
    • Llaves Foráneas (Foreign Keys): Son atributos que crean una relación entre dos entidades. En un ERD, se muestran como una copia de la llave primaria de una entidad en otra entidad.
    • Cardinalidad: Indica el número de instancias de una entidad que pueden estar asociadas con una sola instancia de otra entidad en una relación. Se representa con notaciones específicas en las líneas de relación.

  9. ¿Qué es una entidad?

    10. Una entidad es un objeto, concepto o cosa del mundo real sobre la cual se necesita almacenar información. Una entidad puede ser cualquier cosa que sea distintiva y pueda ser identificada de manera única. En una base de datos, una entidad se representa como una tabla.

    Ejemplos de entidades incluyen:

    • Personas (empleados, clientes)
    • Objetos (productos, vehículos)
    • Eventos (pedidos, transacciones)

  10. ¿Qué es un atributo y sus tipos?

    11. Un atributo es una propiedad o característica de una entidad. En una base de datos, los atributos se representan como columnas en una tabla.

    Tipos de Atributos:

    • Simples: No se pueden dividir en sub-partes más pequeñas. Ejemplo: Nombre, Edad.
    • Compuestos: Se pueden dividir en sub-partes más pequeñas, que representan componentes más básicos con significados independientes. Ejemplo: Nombre Completo que puede dividirse en Nombre y Apellido.
    • Derivados: Su valor puede derivarse de otros atributos. Ejemplo: Edad, que se puede calcular a partir de la Fecha de Nacimiento.
    • Multivaluados: Pueden tener múltiples valores para una sola entidad. Ejemplo: Teléfonos de un empleado.
    • Clave: Un atributo o conjunto de atributos que identifican de manera única a una entidad dentro de un conjunto de entidades. Ejemplo: ID de empleado.

  11. ¿Qué es una relación?

    12. Una relación es una asociación entre dos o más entidades. Las relaciones se utilizan para mostrar cómo las entidades están conectadas entre sí dentro de una base de datos. En un diagrama ER, las relaciones se representan mediante rombos conectados a las entidades correspondientes.

    • Uno a Uno (1:1): Una instancia de una entidad está asociada con una sola instancia de otra entidad y viceversa.

      • Ejemplo: Un empleado tiene un número de seguro social.

    • Uno a Muchos (1): Una instancia de una entidad está asociada con múltiples instancias de otra entidad.

      • Ejemplo: Un cliente puede hacer muchos pedidos.

    • Muchos a Muchos (N): Múltiples instancias de una entidad están asociadas con múltiples instancias de otra entidad.

      • Ejemplo: Los estudiantes pueden inscribirse en muchos cursos y los cursos pueden tener muchos estudiantes.

    Las relaciones también pueden tener atributos propios, conocidos como atributos de relación, que describen las propiedades de la relación entre entidades.

  12. ¿Qué son las llaves primarias, foránea y sus características?

    13. Llave Primaria (Primary Key):

    • Definición: Una llave primaria es un atributo o un conjunto de atributos que identifica de manera única a cada registro en una tabla.

    Características:

    • Unicidad: Cada valor de la llave primaria debe ser único en la tabla.
    • No Nulo: Ningún valor de la llave primaria puede ser nulo.
    • Inmutabilidad: Idealmente, el valor de la llave primaria no debería cambiar una vez que ha sido asignado.
    • Identificación Única: Permite identificar de manera única cada registro en una tabla.
    • Ejemplo: En una tabla de empleados, el ID de Empleado puede ser la llave primaria.

    Llave Foránea (Foreign Key):

    • Definición: Una llave foránea es un atributo o conjunto de atributos en una tabla que se refiere a la llave primaria de otra tabla, creando una relación entre las dos tablas.

    Características:

    • Referencial: Apunta a la llave primaria en otra tabla, estableciendo una relación entre las dos tablas.
    • Integridad Referencial: Asegura que los valores en la columna de la llave foránea correspondan a los valores de una columna en la tabla referenciada (normalmente una llave primaria).
    • Ejemplo:En una tabla de pedidos, el ID de Cliente puede ser una llave foránea que se refiere al ID de Cliente en la tabla de clientes.

  13. ¿Qué son las reglas del Negocio?

    • Definición: Las reglas del negocio son declaraciones que definen o restringen algún aspecto del negocio. Describen las operaciones, definiciones y restricciones que se aplican a una organización con el propósito de alcanzar sus objetivos.
    • Propósito: Aseguran que el negocio funcione de manera eficiente y consistente.

    Tipos:

    • Reglas de Integridad: Garantizan la precisión y consistencia de los datos. Ejemplo: Un pedido debe tener al menos un producto.
    • Reglas de Validación: Aseguran que los datos ingresados cumplan ciertos criterios. Ejemplo: El descuento no puede ser mayor al 50%.
    • Reglas de Procesos: Definen cómo deben llevarse a cabo ciertas tareas o procesos. Ejemplo: Un cliente debe ser aprobado antes de poder hacer un pedido.
    • Ejemplo: Un empleado no puede ser asignado a más de un proyecto a la vez.

  14. ¿Qué es la cardinalidad o mapeo?

    15. La cardinalidad en bases de datos describe el número de instancias de una entidad que pueden o deben estar asociadas con instancias de otra entidad.

    Tipos de Cardinalidad:

    • Uno a Uno (1:1): Una instancia de la primera entidad se relaciona con una sola instancia de la segunda entidad, y viceversa.
    • Uno a Muchos (1): Una instancia de la primera entidad se relaciona con muchas instancias de la segunda entidad, pero cada instancia de la segunda entidad se relaciona con una sola instancia de la primera entidad.
    • Muchos a Muchos (N): Muchas instancias de la primera entidad se relacionan con muchas instancias de la segunda entidad.

    Ejemplo:

    • 1:1: Un pasaporte es emitido a un solo ciudadano, y cada ciudadano tiene un solo pasaporte.
    • 1:N:Un autor puede escribir varios libros, pero cada libro tiene un solo autor.
    • N:M:Los estudiantes pueden inscribirse en varios cursos y cada curso puede tener varios estudiantes.
    • Importancia: La cardinalidad ayuda a definir las relaciones entre entidades en un modelo de datos, asegurando que los datos se almacenen y accedan de manera correcta y eficiente.
